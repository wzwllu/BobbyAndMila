## 架构概览

传统三层架构：Controller → Service → Repository，MVC 模式，分层清晰。但 Controller 层混合了展示逻辑与数据处理逻辑，Service 层承担了过多职责。

---

## 严重风险（P0 — 必须立即修复）

### 1. 用户名枚举漏洞
**位置：** `UserService.java:234-239`
登录时先查 `findByUsername()`，存在与不存在返回不同错误信息（"用户不存在" vs "密码错误"）。攻击者可遍历枚举有效用户名。
**修复：** 统一返回"用户名或密码错误"。

### 2. CSRF 完全禁用
**位置：** `SecurityConfig.java:62`
`.csrf().disable()` 使所有 POST/PUT/DELETE 请求无 CSRF 令牌校验，易受跨站请求伪造攻击。

### 3. 内存分页导致 OOM
**位置：** `TaskService.java:248`, `AdminController.java:331`, `WorkerController.java:227` 等多处
所有过滤和分页在 Java 内存中完成（`findAll().stream().filter().collect()`），不适用于任何规模的数据。
**修复：** 所有查询条件下沉到数据库层（JPQL Specification / QueryDSL）。

### 4. 余额扣减非原子操作（竞态条件）
**位置：** `TaskService.java:102-118`
先查余额再扣减非原子操作，高并发下可超额消耗积分。需要 `@Version` 乐观锁或 `SELECT ... FOR UPDATE`。

### 5. 密码字段 JSON 序列化泄露风险
**位置：** `User.java`
实体使用 `@Data` 包含 `@ToString`，若在 Controller 层序列化 User 对象，密码将被明文返回。

## 各层详细评价

### 实体层
- `@Data` 在 JPA 实体上存在风险（`@ToString`/`@EqualsAndHashCode` 在懒加载循环引用时易异常）
- Application 中状态默认值在 `@PrePersist` 设置，而 Project 在字段声明设置，风格不一致

### 枚举层
- 设计规范，Javadoc 完整。ProjectType 缺少 Javadoc

### DTO 层
- **所有 Request DTO 缺少 Bean Validation 注解**，校验逻辑散落在各 Service 中
- `TaskStatsResponse.projectType` 声明为 String，失去类型安全

### Repository 层
- `TaskExecutionRepository.deleteByCreatedAtBefore()` 缺少 `@Transactional`
- 多处 Repository 方法接受 `User` 实体而非 `userId`，导致调用方需加载完整实体

### Service 层
- 职责过重，混合了业务逻辑、校验逻辑、数据转换逻辑
- `TaskService` 为问题集中区（内存分页、竞态条件、精度丢失）

### Controller 层
- 职责不清晰，混合展示逻辑与数据处理
- 多处在事务外访问懒加载字段
- 分页代码重复

### Config 层
- 安全配置薄弱（CSRF 关闭、FrameOptions 禁用、无密码策略）
- `DataInitializer` 使用 `System.out.println` 输出密码
- 无 `@Order` 保证多个 CommandLineRunner 的执行顺序

### 异常层
- `BusinessException` 设计简洁合理
- 全局异常处理对 HTTP 状态码区分不足，且泄露内部信息

## 关键改进建议（按优先级）

### 立即
1. 修复用户名枚举漏洞
2. 启用 CSRF 保护
3. 重构所有内存分页为数据库层分页
4. 余额扣减加乐观锁
5. 密码字段加 `@ToString.Exclude`

### 短期
6. 增加 `spring-boot-starter-validation` + 全量 DTO 校验注解
7. 使用 `@EntityGraph`/`JOIN FETCH` 修复 N+1 查询
8. 使用 `RoundingMode.HALF_UP` 处理积分精度
9. 为异常添加精确的 HTTP Status 映射
10. 编写核心 Service 层单元测试

### 中期
11. 引入 MapStruct 或手动 DTO-Entity 映射层
12. 抽取通用分页工具类
13. 日志改为 Slf4j Logger，配置轮转
14. 升级 Java 17+
15. 引入 CI/CD 流水线，集成自动测试与代码扫描
