# PMP 项目管理系统 — 功能说明文档

**文档版本**：2.0
**最后更新**：2026-07-08

## 1. 系统概述

PMP（Project Management Platform）是一个基于 Spring Boot 的项目管理系统，采用 B/S 架构。系统分为管理端和工作端两个角色入口，支持项目管理、任务分配、积分计费与消耗等核心业务流程。

### 1.1 角色定义

| 角色 | 说明 | 默认账号 |
|------|------|---------|
| ADMIN | 系统管理员，管理端 | `admin / admin123`（首次启动自动创建） |
| WORKER | 工作人员，工作端 | 由管理员在人员管理中创建 |

### 1.2 技术栈

| 技术 | 版本 |
|------|------|
| Java | 8 |
| Spring Boot | 2.7.18 |
| Spring Security | 5.7.11 |
| Spring Data JPA | — |
| H2 Database | 文件模式（./data/pmp） |
| Thymeleaf | — |
| Maven | — |
| Tomcat | 9.0.107 |

## 2. 功能模块

### 2.1 认证模块

| 功能 | 说明 | 访问路径 |
|------|------|---------|
| 登录 | 用户名/密码登录，根据角色自动跳转 | `/login` |
| 登出 | 退出当前会话 | `/logout` |
| 首页跳转 | ADMIN 角色跳转 `/admin`，WORKER 角色跳转 `/worker` | `/` |

- 密码使用 BCrypt 加密存储
- 基于 Session 的认证机制
- 菜单和接口均受角色权限控制

### 2.2 管理端功能

管理端根路径：`/admin`

#### 2.2.1 项目管理

| 功能 | 请求方式 | 路径 |
|------|---------|------|
| 查看项目列表 | GET | `/admin/projects` |
| 创建项目 | POST | `/admin/projects` |
| 删除项目 | DELETE | `/admin/projects/{id}` |

**项目属性说明：**

| 字段 | 说明 | 可选值 |
|------|------|--------|
| 项目名称 | 必填 | 任意字符串 |
| 项目类型 | 必填 | `RATE_BASED`（单价计费）、`POINT_CONSUMING`（积分消耗） |
| 单价 | 单价计费类型必填 | 正数（元/个） |
| 重复方式 | 必填 | `NONE`（不重复）、`DAILY`（每日）、`WEEKLY`（每周） |
| 每周重复日 | 每周重复类型时填写 | 1（周一）~ 7（周日） |
| 消耗积分 | 积分消耗类型必填 | 正数（每次消耗的积分数） |

#### 2.2.2 人员管理

| 功能 | 请求方式 | 路径 |
|------|---------|------|
| 查看用户列表 | GET | `/admin/users` |
| 创建用户 | POST | `/admin/users` |

- 创建的用户默认角色为 WORKER
- 用户名全局唯一
- 管理员账号不可删除

#### 2.2.3 任务分配

| 功能 | 请求方式 | 路径 |
|------|---------|------|
| 查看分配列表 | GET | `/admin/assignments` |
| 新建分配 | POST | `/admin/assignments` |
| 取消分配 | PUT | `/admin/assignments/{id}/cancel` |

**分配属性：**

| 字段 | 说明 |
|------|------|
| 项目 | 选择已创建的项目 |
| 工作人员 | 选择已创建的工作端用户 |
| 开始日期 | 分配开始日期 |
| 结束日期 | 可选，分配截止日期 |
| 状态 | `ACTIVE`（进行中）、`CANCELLED`（已取消） |

#### 2.2.4 积分管理

| 功能 | 请求方式 | 路径 |
|------|---------|------|
| 查看积分交易记录 | GET | `/admin/points` |
| **调整用户积分** | **POST** | **`/admin/points/adjust`** |

**管理员积分调整（本次新增）**：

- 选择目标用户
- 输入调整积分数（正数）
- 选择"增加积分"或"扣除积分"
- 填写调整原因（可选）
- 系统自动记录交易

> 示例：给 user1 增加 100 积分 → 积分余额增加 100，交易记录中记为 EARN 类型

### 2.3 工作端功能

工作端根路径：`/worker`

#### 2.3.1 任务管理

| 功能 | 路径 |
|------|------|
| 查看我的任务 & 完成记录 | `/worker/tasks` |
| 完成任务 | POST `/worker/tasks/complete` |

**完成任务流程：**
1. 页面显示分配给当前用户的任务列表
2. 点击"完成任务"按钮
3. 输入完成数量
4. 确认完成
   - 如果是单价计费项目：`获得积分 = 单价 × 完成数量`
   - 系统自动创建 TaskExecution 和 PointsTransaction 记录

**今日任务页面包含：**
- 分配的任务列表（项目名称、日期范围、状态）
- 当前积分余额
- 历史完成记录

#### 2.3.2 积分消耗

| 功能 | 路径 |
|------|------|
| 积分管理页面 | `/worker/points` |
| 消耗积分 | POST `/worker/points/consume` |
| 查看余额 | GET `/worker/points/balance` |

**消耗积分流程：**
1. 页面顶部显示当前积分余额
2. 选择消耗积分的项目
3. 输入消耗数量
4. 确认消耗
   - 系统检查余额是否充足
   - 扣除积分，记录 PointsTransaction

## 3. 用户操作流程

### 3.1 管理员完整操作流程

```
登录 → 首页
├── 项目管理 → 创建项目 → 查看列表 → 删除项目
├── 人员管理 → 创建工作端账号 → 查看列表
├── 任务分配 → 新建分配（选项目+选人+设日期）→ 查看列表 → 取消分配
└── 积分管理 → 查看交易记录 → 手动调整积分（选用户+设数量）
```

### 3.2 工作端完整操作流程

```
登录 → 首页
├── 任务管理 → 查看今日任务 → 完成任务 → 查看历史记录
└── 积分管理 → 查看余额 → 选择项目消耗积分
```

## 4. 积分规则

### 4.1 积分获得

- **触发条件**：工作人员完成单价计费类项目任务
- **计算公式**：`获得积分 = 项目单价 × 完成数量`
- **记录方式**：创建 `PointsTransaction` 记录，type=EARN

### 4.2 积分消耗

- **触发条件**：工作人员在积分管理页面选择项目消耗积分
- **限制条件**：消耗数量不得超过当前积分余额
- **记录方式**：创建 `PointsTransaction` 记录，type=CONSUME

### 4.3 管理员调整

- **正数**：增加积分，type=EARN
- **负数**：扣除积分，type=CONSUME

### 4.4 余额计算

```
积分余额 = SUM(EARN.amount) - SUM(CONSUME.amount)
```

## 6. 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 积分不足 | 消耗积分时提示"积分不足"，阻止操作 |
| 用户名重复 | 创建用户时提示"用户名已存在" |
| 用户/项目/分配不存在 | 抛出 BusinessException，前端显示友好提示 |
| 权限不足 | Spring Security 拦截，返回 403 |
| 管理员调整负数 > 余额 | 允许，可通过管理员调整个别修复 |
